从createApp入手

概述

代码位置在packages/runtime-core/src/apiCreateApp.ts

其中，createApp函数是一个工厂函数，用于创建一个Vue实例，并返回一个包含Vue实例和其他API方法的App对象。createApp函数的实现主要包括创建上下文对象、安装插件、注册组件和指令、挂载应用程序等步骤。

App接口定义了一系列用于操作Vue实例和应用程序的方法和属性，包括version属性、config属性、use方法、mixin方法、component方法、directive方法、mount方法、unmount方法和provide方法等。这些API方法和属性可以通过createApp函数创建的App对象进行访问和调用。

此外，代码中还定义了一些类型和常量，如Plugin类型、AppContext接口、RuntimeCompilerOptions接口、OptionMergeFunction类型、NormalizedPropsOptions类型、ObjectEmitsOptions类型、version常量等。

AppContext

AppContext中有optionsCache 、propsCache 、 emitsCache

这些cache的作用是为了优化Vue.js 3.x在组件选项合并、props选项归一化和emits选项归一化等过程中的性能表现，避免重复计算和重复创建对象。

具体来说，optionsCache用于缓存组件选项合并后的结果，避免重复合并；propsCache用于缓存组件props选项归一化后的结果，避免重复归一化；emitsCache用于缓存组件emits选项归一化后的结果，避免重复归一化。

这些缓存是使用JavaScript的WeakMap类型实现的，WeakMap是一种弱引用类型，当缓存的对象被垃圾回收时，缓存中对应的键值对也会被自动删除，从而避免了内存泄漏的问题。

需要注意的是，由于每个应用程序实例都可能具有不同的全局混入和选项合并策略，因此每个应用程序实例都有自己的缓存，避免了不同应用程序之间的干扰。

归一化

在Vue.js中，归一化（Normalization）是指将组件选项中的props和emits选项转化成统一的格式，以便在组件实例化和渲染过程中更方便地使用。

具体来说，归一化props选项是将其转化为一个包含各个props属性配置的对象，其中每个属性都包含了属性类型、默认值、是否必需等信息，这样组件实例在访问props属性时，就可以直接通过访问该对象的属性来获取对应的值，而无需再进行额外的处理。

归一化emits选项是将其转化为一个包含各个事件名称的数组或对象，这样组件实例就可以直接通过访问该数组或对象来触发相应的事件。

归一化过程是在Vue.js编译模板时进行的，主要是通过解析组件选项中的props和emits选项，生成对应的配置对象或数组，然后在组件实例化时将其作为组件实例的属性进行绑定。

归一化可以使Vue.js在处理组件选项时更加简单和高效，同时也能够提高组件的可维护性和可读性。

complierOptions

@vue/compiler-dom 是 Vue.js 编译器的一个模块，用于将模板编译成渲染函数，以供 Vue.js 运行时使用。compilerOptions 属性可以用来设置编译器的一些选项，例如支持的指令、标签、自定义元素等等。这些选项可以影响 Vue.js 应用程序的性能、功能和行为。

NormalizedPropsOptions

NormalizedPropsOptions 是 TypeScript 类型定义中的一个接口，用于描述 Vue.js 组件的 props（属性）配置选项的类型。这些选项包括 type、required、default、validator 等属性，用于定义组件的属性的类型、是否必须、默认值、验证函数等信息。

在 Vue.js 组件开发中，props 是组件的输入数据，用于从父组件向子组件传递数据。NormalizedPropsOptions 接口可以用于指定组件的属性的类型、默认值、验证函数等选项，以便在编译时对组件进行类型检查和验证，从而提高组件的健壮性和可维护性。

接受两个参数：一个是 render，表示应用程序的根渲染函数；另一个是可选的 hydrate，表示应用程序的根水合函数。返回值是一个函数，用于创建 Vue.js 应用程序实例。

函数返回的创建应用程序实例的函数接受两个参数：一个是 rootComponent，表示应用程序挂载的根组件；另一个是可选的 rootProps，表示根组件的 props。

函数内部首先检查 rootComponent 是否是函数类型，如果不是，则将其转换为一个包含了其所有属性和方法的对象。然后检查 rootProps 是否是对象类型，如果不是，则将其设置为 null。

接着，函数创建一个应用程序的上下文对象，并新建一个空的 Set 对象用于存储已安装的插件。然后初始化一个变量 isMounted，表示应用程序是否已经挂载。

接下来，函数返回一个包含了一系列方法和属性的对象 app，用于操作和管理应用程序。其中包括了 use、mixin、component、directive、mount、unmount、provide 等方法，用于安装插件、混入选项、注册组件、注册指令、挂载应用、卸载应用、提供依赖注入等操作。

在 mount 方法中，函数首先检查应用程序是否已经挂载，如果没有，则创建根组件的 vnode 并将其挂载到根元素上。如果应用程序已经挂载，则输出警告信息。

在 unmount 方法中，函数检查应用程序是否已经挂载，如果已经挂载，则卸载应用程序。如果应用程序没有挂载，则输出警告信息。

在 provide 方法中函数首先判断提供的键是否已经存在于应用程序上下文的 provides 属性中，如果存在则输出警告信息。然后将键值对存储到 provides 属性中，并返回应用程序实例。

在函数的最后，如果开启了兼容模式，则调用 installAppCompatProperties 方法安装应用程序的兼容属性。

总体来说，这段代码定义了一个 Vue.js 3 应用程序的创建 API，提供了一系列的方法和属性用于操作和管理应用程序。它的实现涉及到了应用程序的初始化、组件注册、插件安装、应用程序挂载、卸载等多个方面，并且兼容了 Vue.js 2 的一些特性。

createApp

createApp 函数的类型定义，它用于创建一个 Vue.js 应用程序实例。具体来说，这个函数接受三个参数：

  1. root: 应用程序挂载的根元素，可以是一个 DOM 元素或一个选择器字符串。
  2. options: 应用程序的基本配置选项，包括组件、指令、插件等。
  3. hydrate: 一个函数，用于将服务端渲染的 HTML 恢复成可交互的 Vue.js 应用程序。

返回值是一个 CreateAppFunction 类型的函数，用于创建一个 Vue.js 应用程序实例。它接受一个 HostElement 类型的参数，表示应用程序挂载的根元素。

在 TypeScript 中，这段代码的含义是：定义一个名为 createApp 的函数，它可以接受三个参数，第一个参数是 root，第二个参数是 options，第三个参数是 hydrate，返回值是一个用于创建 Vue.js 应用程序实例的函数 CreateAppFunction。同时，它也明确了 hydrate 参数的类型为 RootHydrateFunction。

use方法的解释

首先，use 方法接受两个参数：要安装的插件和插件的选项。如果插件已经安装过，会在开发模式下输出警告信息，并直接返回应用程序实例。否则，检查插件是否是一个对象并且具有 install 方法。如果是，则将插件添加到已安装插件的 Set 集合中，并调用插件的 install 方法，将应用程序实例和插件选项作为参数传递进去。如果插件不是一个对象，而是一个函数，则将其添加到已安装插件的 Set 集合中，并直接调用该函数，将应用程序实例和插件选项作为参数传递进去。如果插件既不是对象也不是函数，则在开发模式下输出警告信息。最后，返回应用程序实例。

createApp解析

  1. createApp(App) 创建了一个 Vue.js 应用程序实例，并将根组件 App 作为参数传递进去。
  2. .use(router) 使用 .use 方法安装了 Vue Router 插件。这里的 router 是一个 Vue Router 实例，它具有一个 install 方法，.use 方法会将其作为参数传递给插件的 install 方法。
  3. .use(ElementPlus, { locale: zhCn }) 使用 .use 方法安装了 Element Plus 插件。这里的 ElementPlus 是一个插件对象，它具有一个 install 方法，.use 方法会将应用程序实例和选项 { locale: zhCn } 作为参数传递给插件的 install 方法。
  4. .use(pinia) 使用 .use 方法安装了 Pinia 插件。这里的 pinia 是一个插件函数，它也可以被 .use 方法安装。.use 方法会将应用程序实例作为参数传递给插件函数。
  5. .mount('#app') 是 Vue.js 应用程序的挂载过程，它将根组件渲染到指定的 DOM 元素中。这里的 #app 是一个 CSS 选择器，表示要将根组件渲染到具有 id="app" 的 DOM 元素中。

在整个初始化的过程中，.mount('#app') 是最后一步，它将应用程序实例挂载到指定的 DOM 元素，让应用程序开始运行。当应用程序挂载完成后，Vue.js 会自动调用根组件的 mounted 生命周期钩子函数，这个函数可以用来执行一些初始化的操作。

总的来说，.mount('#app') 是 Vue.js 应用程序的最后一步，它将应用程序实例与 DOM 元素关联起来，使得应用程序能够在页面上正常运行。
